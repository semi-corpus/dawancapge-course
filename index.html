GIT

Introduction :

Un système de gestion de versions (VCS en anglais pour Version Control System ou aussi SCM pour Source Content Management), ou encore système de versionning, est un système qui enregistre toutes les modifications apportées à une liste de fichiers. C’est un système qui permet de suivre précisément l’évolution du contenu des fichiers.

Le suivi des fichiers est effectué par sauvegarde des modifications. En effet, lorsque vous avez fini de modifier un fichier, vous allez indiquer que vous avez terminé ce travail et pour quelles raisons vous l’avez fait. Les modifications effectuées sur le fichier seront sauvegardées par le système grâce au commit.

Un système de versionning permet donc : de revenir très facilement en arrière et à n’importe quel endroit de l’historique; d'obtenir différentes informations concernant les lignes de code grâce aux messages que les développeurs indiquent lorsqu’ils effectuent des modifications; simplifier la maintenance des applications et l’intégration de nouveaux collaborateurs; la facilité de collaboration entre développeurs; de travailler en mode déconnecté avec son dépôt tout en gardant des fonctionnalités avancées de collaboration lorsqu’on est connecté.


Mémo Commandes :


  • git init : Initialise
  • git config user.name monNom : Défini le nom de l'utilisateur
  • git config user.email mon@email.com : Défini le mail de l'utilisateur
  • git status : Affiche les changements non commit
  • git status -s : Comme status, mais en plus court
  • git add nomFichier.txt : Ajoute un fichier à scanner
  • git add --all : Ajoute tout les nouveaux fichiers à scanner
  • git commit : Envoi sur le repo/ouvre l'éditeur pour mettre un commentaire
  • git commit --amend -m "monComentaireModifié" : Modifie le commentaire
  • git commit -m "Mon commentaire": Ajoute un commentaire et commit (mais doit être add au préalable)
  • git commit -am "Mon commentaire": Ajoute un commentaire et envoi en commit (sans besoin de add)
  • git reset hard : Reset tout les commits
  • git help : Aide git
  • git help maCommande : Aide git sur une commande spécifiée
  • git rm maPage.txt : Supprime un fichier du repo
  • git checkout --maPage.txt : Récup un fichier du repo
  • git vim .gitignore : Ignorer les fichiers à ne pas scanner/à ne pas envoyer sur le repo
  • git vim .gitignore : Ignorer les fichiers à ne pas scanner/à ne pas envoyer sur le repo
  • git ls-tree -r master : Scan et afficher les dossier et fichiers de la branche spécifié
  • git log --oneline : Affiche l'historique des commits (avec leurs commentaires)
  • git log -4 : Affiche les 4 derniers commit
  • git log --stat -3 : Affiche le détail de commit c-3
  • git log --author="nomUtilisateurRecherché" : Affiche tout les commit de l'utilisateur spécifié
  • git log --oneline --before="2019-02-15" : Affiche tout les commitavant une date spécifiée
  • git diff : Affiche les lignes éditées
  • git --word-diff : Affiche les mots éditées
  • git diff xxxxx(ID commit + ancien) yyyy(ID commit + récent) monFichier.txt : Affiche les éditions entre 2 commits
  • git show : Affiche la version du commit (si tag annoté existant)
  • git tag V1.0 : Ajouter un tag au dernier commit
  • git tag V1.0 -d : Supprimer un tag
  • git diff V0.9 V1.0 : Affiche les différences entre deux commits tagués
  • git tag -a versionClient : Ajouter un tag au dernier commit avec plus d'info
  • git tag -a monTag xxxx(ID commit) -m "monCommentaire" : Ajouter un tag au dernier commit avec plus d'infos
  • git tag --list "v1*" : Affiche les tags commençant par la(/les) lettre(s) spécifiées
  • git branch -v : Affiche les branches et leur id
  • git branch dev : Créer une branche nommé dev (exemple). Nous restons pour le moment sur la branche master.
  • git checkout dev : Passer sur la branche spécifiée
  • git checkout -b dev : Créer une branche ET passer dessu.
  • git log --oneline --graph --all : Affiche graphiquement l'historique des commits, des branches, du chemin du head
  • git checkout xxxx(ID commit) -b NouvelleBrancheSurAncienCommit : Créer une nouvelle branche à partir d'un ancien commit spécifié
  • (positionné sur la master) git merge dev : Fusionne et commit la branche spécifiée avec la master
  • (positionné sur une branche différente que celle spécifiée) git nomDeMaBranche -d : supprimer la branche spécifiée
